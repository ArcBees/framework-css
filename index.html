Full width button

If you want your button to act like a block element, add a .btn_wide class.
The display property will be changed to block instead of inline-block and it will takes the full width of his parent container

Let's say you got a button and for a design matter, you want the text to be on more than one line.
Add a class .btn_content right next to the .btn class and the text you want to be on the second line in a span element.

error

If you want to change your input field to an error state, add the class .input_error to the input.

required

If your input field is required to complete the form, wrap inside a container div.input .input_required your input and an icon.input_required__required

Add a class .required to a label if you want the required * to appear next to the label instead of inside the input field.

Form

Default form

By default the field of the form will stack on top of one another with the label on top of the label
Go

Inline fields

If you want your label and input to be displayed on the same line, you just have to add a inline_fields class to the form element and every child of the div.field will be displayed as inline-block. You should wrap your input into a div though. ie9 fix
Go

Inline even fields

If you want your label and input to be displayed on the same line, but you want the input to fill the gap left of the width of the field, simply add the inline_fields_even class the the form element. You probably want to add a width to the label so that all the input start at the same spot.
